# ESP8266 & ArduCAM OV2640: Motion Detection & Tracking

Цей проєкт реалізує систему відеоспостереження на базі мікроконтролера ESP8266 та камери OV2640 (через модуль ArduCAM). Головна особливість системи — **усі обчислення комп'ютерного зору (детекція та трекінг) виконуються безпосередньо на ESP8266**, незважаючи на жорсткі обмеження оперативної пам'яті та процесорного часу. Браузер клієнта використовується виключно для візуалізації отриманих даних.

## 🚀 Основні можливості

* **Режим трансляції (JPEG Stream):** Стандартний стрімінг відео в роздільній здатності від 160x120 до 640x480.
* **Режим детекції руху (Motion Detection):** Аналіз кадру на ESP8266, виділення рухомих об'єктів рамкою (Bounding Box) та передача ЧБ-кадру разом з координатами в браузер.
* **Режим слідкування (Tracking):** Фіксація на конкретному об'єкті за кліком у веб-інтерфейсі. Камера "супроводжує" об'єкт, динамічно оновлюючи його координати.
* **Оптимізація пам'яті:** Використання статичних буферів (`frameA`, `frameB`) для уникнення фрагментації heap-пам'яті мікроконтролера під час безперервної роботи.

## 🧠 Детальний опис алгоритмів

Оскільки ESP8266 не здатний запускати складні нейромережі чи алгоритми на кшталт Haar Cascades, тут застосовано максимально оптимізований під залізо математичний підхід.

### Підготовка даних (YUV Extraction)
Для аналізу зображення повноцінний RGB не потрібен — достатньо чорно-білого зображення. Щоб не витрачати ресурси на конвертацію JPEG, камера переводиться в сирий режим **YUV422**. 
Алгоритм "на льоту" (через SPI) відкидає байти кольору (Cb/Cr) і записує в буфер лише Y-канал (яскравість). Таким чином, кадр 160x120 займає рівно 19 200 байт пам'яті.

### Алгоритм детекції руху (Frame Differencing)
Базується на знаходженні різниці між поточним та попереднім кадрами. 

1. **Попіксельне порівняння:** Алгоритм проходить циклом по масиву пікселів. Якщо модуль різниці яскравості між поточним і попереднім пікселем більший за поріг (`THRESH = 30`), піксель позначається як "рухомий".
2. **Формування рамки (Bounding Box):** Під час перевірки алгоритм запам'ятовує крайні ліві/праві та верхні/нижні координати рухомих пікселів, що автоматично формує прямокутник навколо об'єкта.
3. **Фільтрація шумів:** Якщо загальна кількість рухомих пікселів менша за мінімум (`MIN_PX = 200`), рух ігнорується.
4. **Захист від різкої зміни освітлення (Real-world кейс):** Увімкнення світла в кімнаті викликає масову зміну пікселів. Алгоритм рахує площу змін: якщо змінилося більше 40% кадру (`MAX_RATIO`), це розцінюється як глобальна зміна освітлення, і "хибна детекція" скидається.

### Алгоритм слідкування (ROI Tracking)
Трекінг є розвитком алгоритму детекції, але замість сканування всього кадру він використовує метод **Region of Interest (Зона інтересу)** для оптимізації та утримання цілі.

1. **Захоплення цілі:** Користувач клікає по об'єкту на Canvas у браузері. JavaScript розраховує координати кліку і відправляє їх на ESP8266 (через `/target`).
2. **Обмеження зони пошуку:** Отримавши координати, ESP8266 перестає сканувати весь кадр 160x120. Натомість обчислення різниці кадрів відбувається *лише* у невеликому вікні навколо попередньої відомої позиції об'єкта (додається `pad = 28` пікселів у всі сторони).
3. **Оновлення позиції:** Якщо в цій локальній зоні виявляється рух, нові екстремальні координати (min X/Y, max X/Y) стають новими координатами об'єкта для наступного кадру.
4. **Переваги:** Це суттєво розвантажує процесор (цикл проходить не 19200 ітерацій, а лише кілька тисяч) і не дає рамці "перестрибнути" на інший рухомий об'єкт на фоні, оскільки фон просто не сканується.

## 📡 Протокол передачі даних (Детекція/Трекінг)
Щоб передати кадр і результати обчислень швидко, використовується кастомний бінарний пакет замість повільного JSON. 
ESP формує пакет із двох частин:
1. **Заголовок (8 байт):** - `[0]`: Прапорець наявності руху (0/1).
   - `[1-2]`: Кількість змінених пікселів (uint16).
   - `[3]`: Прапорець валідності обмежувальної рамки (0/1).
   - `[4-7]`: Координати рамки (`X`, `Y`, `Width`, `Height`).
2. **Корисне навантаження (19 200 байт):** Сирий масив Y-каналу (Grayscale пікселі).

На стороні клієнта JavaScript розпаковує цей бінарник (`Uint8Array`) і малює кадр на HTML5 Canvas разом з рамками за один прохід.

## ⚙️ Налаштування та запуск

1. Підключіть ArduCAM OV2640 до ESP8266 за стандартною схемою SPI + I2C. Пін `CS` налаштований на `GPIO 16`.
2. Завантажте скетч через Arduino IDE.
3. Підключіться до Wi-Fi точки доступу:
   - **SSID:** `ESP8266_Cam_Pro`
   - **Пароль:** `12345678`
4. Відкрийте браузер за адресою `http://192.168.4.1` (стандартна IP-адреса SoftAP для ESP).
